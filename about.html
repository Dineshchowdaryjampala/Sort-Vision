Sorting Algorithms

This page provides an overview of different sorting algorithms, including their examples, and time & space complexity.

Bubble Sort

Bubble Sort is a simple sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. The process is repeated until the list is sorted.

Example: [5, 3, 8, 1] → [3, 5, 8, 1] → [3, 5, 1, 8] → [3, 1, 5, 8] → [1, 3, 5, 8]

Selection Sort

Selection Sort repeatedly finds the smallest element in the list and moves it to the correct position. It has a time complexity of O(n²) and is not efficient for large datasets.

Example: [5, 3, 8, 1] → [1, 3, 8, 5] → [1, 3, 5, 8]

Insertion Sort

Insertion Sort builds the sorted list one item at a time. It is efficient for small data sets but inefficient for large ones.

Example: [5, 3, 8, 1] → [3, 5, 8, 1] → [3, 5, 8, 1] → [1, 3, 5, 8]

Merge Sort

Merge Sort follows a divide-and-conquer strategy. It splits the list into smaller parts, sorts them recursively, and then merges them back together.

Example: [5, 3, 8, 1] → ([5, 3], [8, 1]) → ([3, 5], [1, 8]) → [1, 3, 5, 8]

Quick Sort

Quick Sort selects a pivot, partitions the array around it, and recursively sorts the subarrays.

Example: [5, 3, 8, 1] → Pivot: 5 → ([3, 1], [8]) → ([1, 3], [5, 8])

Time & Space Complexity

Algorithm Best Case Average Case Worst Case Space Complexity
Bubble Sort O(n) O(n²) O(n²) O(1)
Selection Sort O(n²) O(n²) O(n²) O(1)
Insertion Sort O(n) O(n²) O(n²) O(1)
Merge Sort O(n log n) O(n log n) O(n log n) O(n)
Quick Sort O(n log n) O(n log n) O(n²) O(log n)
